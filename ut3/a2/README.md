# UT3-A2 Elige tu IDE ideal

Actividad diseñada para escenario telemático

## Objetivos
- Analizar y comparar distintos entornos de desarrollo.  
- Reflexionar sobre los criterios que influyen al elegir un IDE.  
- Justificar una elección técnica de forma razonada.



## Contexto
Imagina que formas parte de un nuevo equipo de desarrollo que va a iniciar un proyecto web o de aplicaciones.  
Antes de empezar, el equipo debe decidir **qué IDE utilizar**.  
Tú eres la persona encargada de **analizar varias opciones y justificar la elección final** ante el grupo.



## Tareas

### Investigación rápida
Busca información sobre **tres IDE distintos** (pueden ser de lenguajes o entornos diferentes).  
Algunos ejemplos:
- Visual Studio Code 
- IntelliJ IDEA
- Eclipse 
- NetBeans 
- PyCharm
- Android Studio
- Replit o Gitpod (IDE online)



### Análisis comparativo

| Criterio | IDE 1: Visual Studio Code | IDE 2: NetBeans | IDE 3: Eclipse |
|----------|---------------------------|-----------------|----------------|
| **Lenguajes principales que soporta** | JavaScript, TypeScript, Python, Java, C/C++, Go, PHP, HTML/CSS… (requiere extensiones para algunos) | Java, PHP, C/C++, HTML5 | Java, C/C++, PHP, JavaScript… (vía plugins) |
| **Peso o consumo de recursos** | Muy ligero, arranque rápido, bajo uso de RAM | Medio; más ligero que Eclipse, pero más pesado que VS Code | Pesado; lento en PCs con poca RAM |
| **Facilidad de instalación** | Instalador pequeño, listo en segundos; casi «listo para usar» | Instalador único; funciona nada más acabar sin plugins extra | Requiere elegir «package» adecuado y añadir plugins después |
| **Funciones destacadas** | IntelliSense, debugger integrado, terminal embebida, Git incorporado, Marketplace con miles de extensiones | Debugger y profiler, constructor visual de GUIs Drag-&-Drop, integración Git/Mercurial, Maven incorporado | Amplio ecosistema de plugins, soporte UML/modelado, debugger avanzado, ideal para proyectos grandes |
| **Interfaz / usabilidad** | Minimalista, altamente personalizable, temas y atajos editables | Interfaz clásica pero más sencilla que Eclipse, amigable para principiantes | Interface algo compleja; requiere tiempo de configuración inicial |
| **Comunidad y soporte** | Comunidad enorme, foros activos y extensiones diarias | Comunidad menor que en sus años áureos, pero aún activa; buena documentación | Comunidad muy grande (años de historia), foros, tutoriales y plugins constantes |
| **Licencia / coste** | Gratuito y código abierto (MIT) | Gratuito y open-source (Apache/Oracle) | Gratuito y open-source (EPL) |


### Conclusión y justificación 

- ¿Cuál de los tres IDE recomendarías para tu grupo de trabajo? 
RESPUESTA: De los tres IDE recomendaría Visual studio code porque es gratuito, personalizable y es eficiente para una gama aplia de lenguajes.
- ¿Por qué crees que se adapta mejor a un proyecto de desarrollo web o de aplicaciones? 
RESPUESTA: Visual studio code se adapta de forma positva a ambos casos. Sin embargo, es particularmente ideal para proyectos web debido a su excelente soporte para tecnologías como HTML, CSS y JavaScript.
- ¿Qué criterio has valorado más (rendimiento, comodidad, lenguaje, integración, etc.)?
RESPUESTA: He valorado varios criterios como la comodidad, es decir si es de fácil instalación, su interfaz es intuitiva, los lenguaje principales con el IDE, si es gratuito en este caso es gratuito, el rendimiento si consume muchos recursos, Visual Studio Code es un IDE ligero, pero su consumo puede aumentar dependiendo de cuantas extensiones tengamos instaladas, procesos en segundo plano y la complejidad del proyecto.
